# Редактор карт (Map Editor)

Окно для визуального редактирования уровней Icarus. Уровень хранится как
`LevelData` (`.asset`), а в сцену спавнится в рантайме через `LevelBuilder`.
Редактор правит **только данные ассета**, сцену не трогает.

Открыть: меню **NGJ → Редактор карт**.

Код: [`Assets/_Project/Scripts/Editor/MapEditorWindow.cs`](../Assets/_Project/Scripts/Editor/MapEditorWindow.cs).

---

## Что такое уровень

`LevelData` ([LevelData.cs](../Assets/_Project/Scripts/Gameplay/LevelData.cs)) — ScriptableObject:

- **Mode** (режим камеры/границ):
  - `Free` — камера следует свободно (XY), смертельна только нижняя линия.
  - `UpOnly` — камера едет только вверх (X залочен). Левый/правый края экрана — **физические стены** (врезаешься, не умираешь), смертельна только нижняя линия.
  - `SingleScreen` — камера статична, смертельны лево/право/верх (+ низ всегда).
- **CameraSize** — половина высоты вьюпорта в юнитах. Задаёт kill-стены в UpOnly/SingleScreen. Камера ≈ как в сцене Game (size 5 + LetterboxCamera 16:9).
- **Start** — точка спавна игрока; в UpOnly/SingleScreen ещё и якорь камеры.
- **Goal** + **GoalRadius** — финиш (солнце), касание зоны = победа.
- **KillY** — высота нижней линии смерти. Камера не опускается ниже неё.
- **TimeScale** — глобальный множитель скорости игры.
- **StartStreamIndex** — легаси-фоллбек: используется для респавна только если уровень
  собран без LevelBuilder. С данными уровня спавн всегда в точке **Start**.
- **Streams[]**, **Hazards[]** — потоки и препятствия.

## Потоки (StreamDef)

Три способа задать траекторию (приоритет: круг → рисованный → генератор формы):

1. **Круг** (`IsCircle`) — замкнутое кольцо. Поле **CircleRadius**; число точек растёт с радиусом (`CirclePointCount`, 8..64).
2. **Рисованный путь** (`CustomPoints`, ≥2 точки) — ручная полилиния. `+ Поток` создаёт именно его (две стартовые точки).
3. **Генератор формы** (`Shape`) — параметрические формы (Line, Arc, Circle, Spiral, …) с полями Size/Size2/Count/Turns/Seed/Reverse.

**Scale (Масштаб)** — равномерный множитель всей траектории, работает для **любого** типа (круг, рисованный, форма). Не меняет ширину (Width). 1 = как есть, 2 = вдвое больше. В рантайме применяется как `transform.localScale` потока, в редакторе/бейке — множит точки. Поле в общей секции «Поток» (есть и в групповой правке нескольких выбранных).

Параметры течения (любой тип):
- **Speed** — скорость течения. Цвет потока **вычисляется из скорости** (синий медленно → красный быстро), вручную не задаётся.
- **SpeedEnd (Скорость в конце)** — линейный разгон **вдоль текущего направления течения**:
  Speed на входе потока → SpeedEnd на выходе (макс достигается, если прошёл путь целиком).
  0 = постоянная скорость. При развороте течения рампа переворачивается → на развороте
  скорость снова падает до Speed и набирается заново. На канвасе стрелки растут к концу.
- **Width** — ширина зоны захвата (рисуется «колбасой» на карте).
- **ActiveDuration / InactiveDuration** — пульс: сек включён / выключен (0 = всегда вкл).
- **PulsePhase (Сдвиг фазы)** — сдвиг пульс-цикла, сек, чтобы несколько мигающих потоков
  гасли вразнобой (как PhaseOffset у конусов горелки). Цикл 2с, соседи 0/1 = противофаза.
- **ReverseInterval** — каждые N сек разворот течения (0 = никогда).
- **Turbulence** — поперечное дрожание (0 = ровный).
- **Grip (Хват к оси)** — притяжение к центральной линии траектории. 3 = дефолт;
  6–10 = «рельсы», игрок не слетает на резких поворотах даже при большой Speed;
  1 = рыхлая «река».
- **CatchRate (Скорость подстройки)** — как быстро скорость игрока подстраивается под
  течение, **независимо** от притяжения к оси. Дефолт **10** = почти мгновенно (~0.15с
  до скорости потока); 0 = как «Хват к оси» (легаси); меньше = вязко, импульс держится.
- **ExitBoost (Импульс выхода)** — множитель скорости игрока в момент складывания
  крыльев внутри потока. 1 = чистое сохранение импульса; 1.3–1.8 = катапульта;
  меньше 1 = вязкий поток, из которого тяжело выстрелить.
- **Z (приоритет)** — слой захвата. Когда несколько потоков накрывают Икара
  одновременно, несёт тот, у кого Z больше; нижние слои игнорируются в зоне
  перекрытия. OLD (рельсы): захватывает один верхний (при равном Z — тот, в чьей
  полосе Икар глубже). NEW (поле): блендятся только потоки верхнего слоя. Все Z=0
  (дефолт) → старое поведение (все потоки участвуют).
- **EndpointReverse (Реверс в конце)** — доехал до конца пути (например, центра спирали) →
  течение разворачивается, скорость игрока сбрасывается до базовой Speed. Игрок ходит
  туда-обратно, каждый раз разгоняясь к выходу. Работает в обоих режимах (OLD/NEW), только
  для незамкнутых путей. Тоггл в инспекторе потока.
- **Reverse** — развернуть направление. Кнопка **«⇄ Развернуть стрелки»** в
  инспекторе потока (и «...все» при группе) — быстрый toggle; работает для любого
  типа траектории, включая рисованные пути.
- **Position / Rotation** — размещение траектории в мире.

## Препятствия (HazardDef)

- **Position** — положение.
- **PatrolTravel** — вектор колебания туда-обратно.
- **PatrolPeriod** — период цикла, сек.
- **Size** — радиус.

## Горелки / конусные лучи (BurnerDef)

Точка-якорь, из которой выходит **любое число конусов** (`ConeDef`). Конус жжёт
Икара, пока он внутри сектора. Шкала жара **не показывается** — её эффект только в
постепенной перекраске Икара в красный. Полное прогорание под лучом — за **1 сек**
(`HeatUpTime`), остывание вне луча — за **2 сек** (`CoolDownTime`); при 100% — та же
смерть, что от препятствия (через `Hazard.Kill()` → респавн, жар сбрасывается).

Горелка: одна `Position` + массив `Cones`. Каждый конус:
- **Angle** — направление, градусы (0 = вправо). Для вращения — стартовый угол.
- **Length** — дальность луча.
- **HalfAngle** — половина раствора, градусы (полный = 2·HalfAngle).
- **Motion** — `Static` / `Rotate` / `Pulse`.
  - `Rotate`: **RotateSpeed** (град/сек, знак = направление).
  - `Pulse`: **OnDuration** (горит) / **OffDuration** (погас) / **PhaseOffset**
    (сдвиг фазы, чтобы конусы мигали вразнобой).

Рантайм-объект `Burner` — полностью кодовый (без префаб-арта): сам строит спрайты
конусов (треугольник с альфа-фейдом к концу луча), крутит/мигает, проверяет попадание
Икара и кормит его `BurnState`. Хитбокс — математика сектора (dist+угол), визуал ровно
по ней. Шкала и перекраска — компонент `BurnState` на префабе Икара (красит все спрайты
тела от родного цвета к красному по уровню жара).

## Молнии Зевса (ZeusDef)

Точка-якорь, которая периодически **электризует выбранные потоки**. Пока поток
заряжен — по нему бегут молнии (визуал), и Икар, пойманный **внутри полосы**
(`Width`) такого потока, получает удар: крылья принудительно складываются и
**блокируются** на время `WingBlockDuration` (глобально в GameConfig, дефолт 1 сек),
тело **мигает жёлтым** на тот же срок. Повторный удар продлевает окно.

Зевс: одна `Position` + список целей по индексам + тайминг.
- **TargetStreams** — индексы потоков (из списка `Streams`), по которым бьёт.
  В инспекторе — чекбоксы на каждый поток уровня.
- **FireInterval (Период)** — длина полного цикла, сек: заряжен, потом тишина.
- **ElectrifyDuration (Электризация)** — сколько секунд потоки заряжены каждый цикл
  (зажимается ≤ периода).
- **PhaseOffset (Сдвиг фазы)** — чтобы несколько узлов били вразнобой.

Рантайм-объект `Zeus` — полностью кодовый (без префаб-арта): по таймеру (как `Pulse`
у конуса) определяет фазу заряда, рисует молнии `LineRenderer`-ом — зигзаг-удар от
якоря к ближайшей точке каждого целевого потока + бегущая по всей длине потока
молния, — проверяет, в полосе ли Икар (через `SampleNearest`/`Width`), и кормит его
`ShockState.Shock()`. Блокировка крыльев живёт в `IcarusController.BlockWings()`;
мигание и пуш длительности — компонент `ShockState` на префабе Икара. Респавн
сбрасывает блок и мигание (`ResetShock`). Тюнинг (длительность блока, цвет мигания) —
в GameConfig, как у горелок.

> Индексы `TargetStreams` не переиндексируются при удалении потока (как и
> `StartStreamIndex`). Удалил поток — проверь цели Зевсов. Out-of-range цели
> молча игнорируются.

## Тулбар

Строка 1 (ассет): пикер `LevelData` · **Новый уровень** · **Копировать** ·
**Сохранить** (пишет ассет на диск; звёздочка = есть несохранённое; правки и без
сохранения видны в Play — кнопка нужна перед коммитом/закрытием Unity).
Строка 2 (элементы): **+ Поток** · **+ Форма ▾** · **+ Круг** · **+ Развилка** · **+ Воронка** · **+ Препятствие** · **+ Горелка** · **+ Зевс**.

- **Новый уровень** — диалог сохранения, создаёт пустой `LevelData` в `Assets/_Project/Levels/`.
- **Копировать** — глубокая копия текущего уровня в новый ассет.
- **+ Поток** — новый поток как рисованный путь (2 точки).
- **+ Форма ▾** — выпадающий список из 19 параметрических функций (линия, дуга,
  спираль, восьмёрка, мёртвая петля, штопор, звезда, сердце, клин-V и т.д.) — создаёт поток
  с выбранной формой; её размеры/витки крутятся в инспекторе, поле «Форма» там же
  (по-русски). «Превратить в рисованный путь» — запечь форму в точки и гнуть руками.
- **+ Круг** — круговой поток (радиус 4 по умолчанию).
- **+ Развилка** — ствол (снизу вверх) + 3 рукава веером (вверх-влево / вверх / вниз-вправо),
  каждый — отдельный рисованный поток с общим origin. Создаётся выделенной группой:
  тащишь — двигается целиком; рукава гнёшь точками; между рукавами перепрыгиваешь крыльями.
- **+ Воронка** — два рукава, оба текут **внутрь** в общий центр (обратное развилке):
  стрелки сходятся, засасывает в провал. Два отдельных потока, общий origin, выделены группой.
  (Одиночная непрерывная V-форма — это «Клин (V)» в «+ Форма ▾».)
- **+ Препятствие** — новое препятствие.
- **+ Горелка** — новая точка-горелка с одним конусом (конусы добавляются в инспекторе).
- **+ Зевс** — новый узел молний (цели и тайминг задаются в инспекторе).

## Канвас

- **Колесо** — зум (к курсору). **Средняя кнопка / ЛКМ по пустому** — панорама.
- **ЛКМ по элементу** — выбрать. Стрим выделяется кликом **по любому месту колбасы**, не только по origin.
- **Shift/Ctrl+клик** — добавить/убрать элемент из выделения (мультивыделение).
  Группа двигается целиком, копируется (Ctrl+C/V) и удаляется (Delete) целиком.
  Клик по уже выделенному без модификатора сохраняет группу (можно тащить всю).
- **Тащить** выбранный элемент — двигать (с группой — всю группу).
- **Поворот мышью** — у выбранного элемента/группы над центром рисуется кольцо с синим
  узлом. Тащи узел по кругу — крутит вокруг origin (одиночный) или центроида (группа).
  Для группы каждый элемент и облетает центр, и доворачивается сам (жёсткий поворот).
  Точный угол одиночного потока всё ещё есть в инспекторе (поле «Поворот»).
- **Линия смерти** таскается: клик по синей линии (где нет других элементов) →
  тащи вверх/вниз. Точное значение — поле KillY в инспекторе уровня.
- **Тащить жёлтую точку** рисованного пути — двигать точку.
- **Тащить конец конуса** (точка на острие, видна когда горелка выбрана одна) —
  поворот + удлинение конуса разом (угол = направление, длина = расстояние).
- **Alt+клик** по канвасу (выбран рисованный путь) — добавить точку в конец.
- **ПКМ по точке** рисованного пути — удалить точку (минимум 2 остаются).
- **Delete / Backspace** — удалить выбранный поток/препятствие (уровень и ассет не трогаются).
- **Ctrl+C / Ctrl+V** — копировать/вставить выбранный поток или препятствие
  (вставка со смещением; повторные Ctrl+V кладут «лесенкой»). **Ctrl+D** — дубликат одним нажатием.

Отрисовка: kill-линия снизу, kill-стены по режиму (от Start, размер от CameraSize), потоки «колбасой» по ширине + стрелки направления (длина ∝ скорость). Цвет колбасы — по скорости; **цвет стрелок — по направлению**: вверх = зелёные, вниз = красные, вбок = жёлтые (плавный градиент по углу). На точке Start рисуется **силуэт Икара в реальном игровом размере** (тело + веер крыльев, размах ~2.2 юнита) — референс масштаба; отдельно не выделяется, двигается вместе со Start. Потоки с разгоном (SpeedEnd > Speed) помечены значком **⚡ у выхода** + «⚡+N» в метке: диск «нагревается» (зелёный → оранжевый → красный) и растёт с величиной разгона — сильнейшие разгоны видно сразу.

## Инспектор (правая панель)

Зависит от выбора:
- Ничего выбрано → параметры **уровня**.
- **Несколько элементов** (Shift/Ctrl+клик) → сводка группы + **групповая правка
  параметров потока** (Speed/Width/пульс/реверс/турбулентность/Grip): ввод значения
  задаёт его всем выбранным, «+/−» сдвигает каждого от его текущего значения,
  прочерк = у выбранных значения различаются. При **ровно 2 выбранных потоках** —
  кнопка **«🔗 Склеить по краям»**: стыкует их в один рисованный путь по ближайшей паре
  крайних точек (нужную сторону разворачивает), параметры берутся от первого; оба
  исходных удаляются. Только незамкнутые пути.
- Поток → параметры потока (круг: Радиус+Реверс+«Превратить в рисованный путь»; рисованный: точки+конвертация+**«Сгладить углы»**; форма: поля генератора). **«Сгладить углы»** (Chaikin corner-cutting) скругляет резкие переходы рисованного пути; жми повторно для большей гладкости (удваивает точки). Есть и групповая **«Сгладить углы (все)»** для нескольких выбранных (рукава развилки/воронки, склейки). Числовые поля Flow имеют кнопки **[+][−]** и русские tooltip'ы. **Любая** форма (включая круг и замкнутые петли) бейкается в рисованный путь: кнопка «Превратить в рисованный путь» → точки гнутся руками, замкнутость сохраняется (`CustomLoop`), гладкость держит Catmull-Rom на рантайме.
- Препятствие → параметры препятствия.
- Горелка → позиция + **список конусов** (каждый: угол/длина/полуугол/движение и
  зависимые поля), кнопки **+ Конус** / **Удалить** на конус, **Удалить горелку**.
- Зевс → позиция + тайминг (период/электризация/сдвиг фазы) + **чекбоксы целей**
  (по потоку на строку), **Удалить Зевса**. На канвасе пунктир от узла к каждому
  целевому потоку.

Все правки идут через `SerializedObject` + `Undo`, ассет помечается dirty.

## Как уровень попадает в игру

1. В сцене объект `GameConfig` ссылается на `LevelBuilder` и стартовый `LevelData` (`_startLevel`).
2. `GameConfig.Awake` (exec order −100) зовёт `LevelBuilder.BuildNow()` — спавнит потоки/препятствия/горелки из данных, отдаёт геометрию `LevelController` (`Configure`).
3. `LevelController` применяет режим/размер камеры/границы, спавнит игрока.

Чтобы сменить уровень: в инспекторе `GameConfig` поле **Start Level** → нужный `.asset`.

## Архитектурные заметки

- Редактор НЕ редактирует сцену/префабы — только `LevelData`.
- Тулбар рисуется ручным `GUI` (не `GUILayout`), т.к. канвас использует `GUI.BeginClip`; мешать две layout-системы в одном `OnGUI` нельзя (десинк GUIClip-стека).
- Мутации массивов отложены через `_pending` (выполняются вне открытых layout-групп).
- Delete перехватывается как **command event** ("Delete"/"SoftDelete"), не как KeyDown — иначе Unity удалил бы выбранный ассет.
- `_level` помечен `[SerializeField]` — переживает domain reload.
- Цвет потока и точки круга считаются общими хелперами в `LevelData`/`StreamShapeBuilder` — едины для редактора и рантайма.
