# Редактор карт (Map Editor)

Окно для визуального редактирования уровней Icarus. Уровень хранится как
`LevelData` (`.asset`), а в сцену спавнится в рантайме через `LevelBuilder`.
Редактор правит **только данные ассета**, сцену не трогает.

Открыть: меню **NGJ → Редактор карт**.

Код: [`Assets/_Project/Scripts/Editor/MapEditorWindow.cs`](../Assets/_Project/Scripts/Editor/MapEditorWindow.cs).

---

## Что такое уровень

`LevelData` ([LevelData.cs](../Assets/_Project/Scripts/Gameplay/LevelData.cs)) — ScriptableObject:

- **Mode** (режим камеры/границ):
  - `Free` — камера следует свободно (XY), смертельна только нижняя линия.
  - `UpOnly` — камера едет только вверх (X залочен), смертельны левый/правый края экрана.
  - `SingleScreen` — камера статична, смертельны лево/право/верх (+ низ всегда).
- **CameraSize** — половина высоты вьюпорта в юнитах. Задаёт kill-стены в UpOnly/SingleScreen. Камера ≈ как в сцене Game (size 5 + LetterboxCamera 16:9).
- **Start** — точка спавна игрока; в UpOnly/SingleScreen ещё и якорь камеры.
- **Goal** + **GoalRadius** — финиш (солнце), касание зоны = победа.
- **KillY** — высота нижней линии смерти. Камера не опускается ниже неё.
- **TimeScale** — глобальный множитель скорости игры.
- **StartStreamIndex** — индекс потока, в который респавнится игрок.
- **Streams[]**, **Hazards[]** — потоки и препятствия.

## Потоки (StreamDef)

Три способа задать траекторию (приоритет: круг → рисованный → генератор формы):

1. **Круг** (`IsCircle`) — замкнутое кольцо. Поле **CircleRadius**; число точек растёт с радиусом (`CirclePointCount`, 8..64).
2. **Рисованный путь** (`CustomPoints`, ≥2 точки) — ручная полилиния. `+ Поток` создаёт именно его (две стартовые точки).
3. **Генератор формы** (`Shape`) — параметрические формы (Line, Arc, Circle, Spiral, …) с полями Size/Size2/Count/Turns/Seed/Reverse.

Параметры течения (любой тип):
- **Speed** — скорость течения. Цвет потока **вычисляется из скорости** (синий медленно → красный быстро), вручную не задаётся.
- **Width** — ширина зоны захвата (рисуется «колбасой» на карте).
- **ActiveDuration / InactiveDuration** — пульс: сек включён / выключен (0 = всегда вкл).
- **ReverseInterval** — каждые N сек разворот течения (0 = никогда).
- **Turbulence** — поперечное дрожание (0 = ровный).
- **Reverse** — развернуть направление.
- **Position / Rotation** — размещение траектории в мире.

## Препятствия (HazardDef)

- **Position** — положение.
- **PatrolTravel** — вектор колебания туда-обратно.
- **PatrolPeriod** — период цикла, сек.
- **Size** — радиус.

## Тулбар

Строка 1 (ассет): пикер `LevelData` · **Новый уровень** · **Копировать**.
Строка 2 (элементы): **+ Поток** · **+ Круг** · **+ Препятствие**.

- **Новый уровень** — диалог сохранения, создаёт пустой `LevelData` в `Assets/_Project/Levels/`.
- **Копировать** — глубокая копия текущего уровня в новый ассет.
- **+ Поток** — новый поток как рисованный путь (2 точки).
- **+ Круг** — круговой поток (радиус 4 по умолчанию).
- **+ Препятствие** — новое препятствие.

## Канвас

- **Колесо** — зум (к курсору). **Средняя кнопка / ЛКМ по пустому** — панорама.
- **ЛКМ по элементу** — выбрать. Стрим выделяется кликом **по любому месту колбасы**, не только по origin.
- **Тащить** выбранный элемент — двигать.
- **Тащить жёлтую точку** рисованного пути — двигать точку.
- **Alt+клик** по канвасу (выбран рисованный путь) — добавить точку в конец.
- **Delete / Backspace** — удалить выбранный поток/препятствие (уровень и ассет не трогаются).

Отрисовка: kill-линия снизу, kill-стены по режиму (от Start, размер от CameraSize), потоки «колбасой» по ширине + стрелки направления (длина ∝ скорость), цвет по скорости.

## Инспектор (правая панель)

Зависит от выбора:
- Ничего выбрано → параметры **уровня**.
- Поток → параметры потока (круг: только Радиус+Реверс; рисованный: точки+конвертация; форма: поля генератора). Числовые поля Flow имеют кнопки **[+][−]** и русские tooltip'ы.
- Препятствие → параметры препятствия.

Все правки идут через `SerializedObject` + `Undo`, ассет помечается dirty.

## Как уровень попадает в игру

1. В сцене объект `GameConfig` ссылается на `LevelBuilder` и стартовый `LevelData` (`_startLevel`).
2. `GameConfig.Awake` (exec order −100) зовёт `LevelBuilder.BuildNow()` — спавнит потоки/препятствия из данных, отдаёт геометрию `LevelController` (`Configure`).
3. `LevelController` применяет режим/размер камеры/границы, спавнит игрока.

Чтобы сменить уровень: в инспекторе `GameConfig` поле **Start Level** → нужный `.asset`.

## Архитектурные заметки

- Редактор НЕ редактирует сцену/префабы — только `LevelData`.
- Тулбар рисуется ручным `GUI` (не `GUILayout`), т.к. канвас использует `GUI.BeginClip`; мешать две layout-системы в одном `OnGUI` нельзя (десинк GUIClip-стека).
- Мутации массивов отложены через `_pending` (выполняются вне открытых layout-групп).
- Delete перехватывается как **command event** ("Delete"/"SoftDelete"), не как KeyDown — иначе Unity удалил бы выбранный ассет.
- `_level` помечен `[SerializeField]` — переживает domain reload.
- Цвет потока и точки круга считаются общими хелперами в `LevelData`/`StreamShapeBuilder` — едины для редактора и рантайма.
